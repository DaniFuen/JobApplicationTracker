# AI Usage Documentation

## Tools Used
- Claude – used for HTML templates and debugging
- ChatGPT – used for brainstorming features and understanding error messages

## Key Prompts
1. "Build complete HTML templates for a job tracker web application with a dark theme"
2. "Add emojis to the sidebar navigation icons to make it look better"
3. "Fix this error: werkzeug.exceptions.BadRequestKeyError: KeyError 'company_id'"
4. "Why is my dashboard showing blank with no data?"
5. "How do I connect my Flask app to MySQL Workbench?"

## What Worked Well 
- AI generated clean HTML templates quickly
- Emojis made the sidebar navigation look more professional
- AI explained every error message clearly and gave exact fixes
- Dark theme CSS matched the design style I was going for

## What I Modified
- Changed database password and credentials to match my local MySQL setup
- Tested and verified each page worked correctly in the browser
- Removed index.html which was conflicting with dashboard.html
- Verified all form field names matched correctly in app.py

## Lessons Learned
- Always test AI generated code before assuming it works
- AI is great for HTML and styling but you still need to understand your own file structure
- Debugging errors is much faster when you can describe the exact error message to AI
- GenAI tools save a lot of time on repetitive code like HTML templates
